# Procurement Summary

## Purpose

This document defines the **approval boundaries, scope limitations, and procurement considerations** for the NovaFuse ERIL Reference Surface. It is designed to help procurement and legal teams evaluate what is being approved and what responsibilities remain with the organization.

---

## What Is Being Approved

### ✅ Public Reference Surface

This repository provides a **complete, verifiable reference implementation** of ERIL (Executable Reference Implementation Language) including:

- **Language specification** - Formal ERIL definition and grammar
- **Working examples** - Minimal ERIL → ERI → Evidence flows
- **Verification tools** - Structural validation and replay capability
- **Documentation** - Complete technical and procedural guidance
- **Schema definitions** - Evidence structure specifications

### ✅ Evaluation Capability

The approved artifacts enable:
- **Independent verification** - Replay without NovaForge involvement
- **Structural validation** - Schema-based artifact checking
- **Process compliance** - Evidence of execution adherence
- **Reference implementation** - Gold standard for ERI development

---

## What Is NOT Being Approved

### ❌ Proprietary Governance Logic

**Explicitly excluded:**
- **Normative specifications** - Complete SHALL/SHOULD/MAY corpus
- **Governance lifecycle** - Authoring → approval → deployment procedures
- **Institutional frameworks** - Responsibility allocation and decision matrices
- **Compliance thresholds** - Scoring weights and decision criteria
- **Trade-secret methods** - Proprietary algorithms and techniques

### ❌ Operational Deployment

**Not included:**
- **Production systems** - Deployed governance platforms
- **Enterprise integration** - Connected organizational systems
- **Managed services** - Ongoing governance operations
- **Support contracts** - Maintenance and update agreements

### ❌ Certification Authority

**Not provided:**
- **Regulatory approval** - Compliance certification
- **Standards conformance** - Official accreditation
- **Legal validation** - Jurisdictional compliance
- **Risk assessment** - Organizational risk analysis

## Scope Limitations

### 🎯 Reference Use Only

**Approved for:**
- **Evaluation and testing** - Understanding ERIL capabilities
- **Reference implementation** - Development guidance
- **Academic research** - Study of executable governance
- **Standards review** - Technical assessment

**Not approved for:**
- **Production deployment** - Live operational use
- **Commercial licensing** - Resale or distribution
- **Regulatory submission** - Compliance certification
- **Mission-critical systems** - Safety-critical deployment

### 📋 Bounded Context

**Context limitations:**
- **Single ERI focus** - Individual governance decisions
- **Micro-task scope** - Limited, well-defined operations
- **Demonstration purposes** - Educational and evaluative use
- **Non-clinical applications** - No medical or safety-critical use
